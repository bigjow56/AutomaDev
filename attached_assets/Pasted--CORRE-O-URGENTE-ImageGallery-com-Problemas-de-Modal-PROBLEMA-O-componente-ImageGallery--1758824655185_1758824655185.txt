# CORREÇÃO URGENTE - ImageGallery com Problemas de Modal

## PROBLEMA:
O componente ImageGallery tem bugs críticos no modal/lightbox que impedem o uso adequado:

1. **Modal ocupa mais espaço que a viewport** - usuário precisa diminuir zoom para ver tudo
2. **Scroll bloqueado na página principal** - só funciona dentro da área da imagem expandida  
3. **Imagem cortada** - não é possível ver a imagem completa
4. **Layout quebrado** - modal não se adapta corretamente ao tamanho da tela

## CAUSAS IDENTIFICADAS:
- Imagem principal sem limites de tamanho (`max-w-full max-h-full` causa overflow)
- Container do modal com height/overflow mal configurados
- Conflito no controle de scroll (usando classList e style ao mesmo tempo)
- Posicionamento relativo em elementos que deveriam ser fixed

## CORREÇÕES NECESSÁRIAS:

### 1. LIMITAR TAMANHO DA IMAGEM:
```css
/* Em vez de: max-w-full max-h-full */
max-w-[85vw] max-h-[70vh]
style={{ maxWidth: 'min(85vw, 1200px)', maxHeight: 'min(70vh, 800px)' }}
```

### 2. CORRIGIR CONTAINER PRINCIPAL DO MODAL:
```jsx
{/* Container com scroll fixo */}
<div className="h-full w-full overflow-y-auto overflow-x-hidden pt-20 pb-4">
```

### 3. SIMPLIFICAR CONTROLE DE SCROLL:
```js
// Só usar style, remover classList
document.body.style.overflow = 'hidden';
document.documentElement.style.overflow = 'hidden';
```

### 4. HEADER E CONTROLES COM POSITION FIXED:
```jsx
{/* Header sempre visível no topo */}
<div className="fixed top-0 left-0 right-0 z-[60] bg-gradient-to-b from-black/90 via-black/70 to-transparent">
```

## ARQUIVO PARA SUBSTITUIR:
O arquivo `components/image-gallery.tsx` precisa ser completamente substituído pela versão corrigida.

## RESULTADO ESPERADO:
- Modal abre no tamanho correto da tela
- Scroll funciona naturalmente no conteúdo  
- Imagens visíveis sem cortes
- Controles sempre acessíveis
- Funciona em qualquer resolução

## URGÊNCIA: ALTA
Este bug impede o uso básico da funcionalidade de galeria de imagens.
